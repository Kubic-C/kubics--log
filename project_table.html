Project (PMUT); find compiled versions (here):

PMUT or P.M.U.T. is the first big project
I had ever worked on, and for me it taught
a lot. The project itself was like a cli
alternative with my additional commands and
other things. I had a lot of fun programming
it and additionally most of what I know of C++
comes from it.

PMUT used opengl with its own in-built console
with in and out buffers. You could render text by pirority
and make your own commands and use them through PMUT.
PMUT had a lot put it into it which when making big projects
made it hard to work on it and find bugs. It also made me
realize that classes are probably not the best for
building most of an appilcation.

And if you're wondering what PMUT stands for it is:
Porter Multi Use Terminal.

Project (PMUT-CMAKE):

PMUT-CMAKE is "crossplatform" version of PMUT.
PMUT-CMAKE was the first project I had made
on a linux-distro specifically Ubuntu. PMUT-CMAKE
in my opinion was a MUCH cleaner version the
original PMUT. And the reason it is called
PMUT-CMAKE if it you did not get it from the
name is becuase it was made with CMAKE

PMUT-CMAKE taught me how to use CMAKE,
the CLI, among a few other things that
you can only learn from using a linux-distro.
PMUT-CMAKE had all if not more features
then the original PMUT.

Project (kubics-console):

kubics-console takes the console from
PMUT and allows you to use it. I have
not learned much from the project. So
I will go over what it can do and what
it will do.

kubics-console is a cross-platform library.
That allows you to directly write and read
to its output buffer and input buffer. You
can port console games to it to allow them
to be cross platform. You can use this instead
of your OS' native console. If you want to
use more then just std::cout to maniuplate your
console, I would suggest kubics-console.
